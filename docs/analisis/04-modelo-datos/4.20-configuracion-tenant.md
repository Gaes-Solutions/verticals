# GaesSoft POS — Modelo 4.20 Configuración tenant

> **Estado:** Cerrado · **Fecha cierre:** 2026-04-27
> **Memoria persistente:** `project_gaes_pos_modelo_4_20_configuracion_tenant.md`
> **Descripción:** Tenant schema: branding, localización, POS, pagos, inventario, plantillas impresión, módulos activos, feature flags, webhooks, API keys, backups

Sub-análisis 4.20 cerrado por Gaby. Bucket de configuración personalizable que NO encaja en otros modelos (4.19 fiscal, 4.18 marketing, 4.6 usuarios).

## Bloque A — `config_branding`
Logo + logo_blanco (fondo oscuro), favicon, colores primario/secundario/acento, fuente, tamaño_base, subdominio_gaessoft (cliente.gaessoft.com), dominio_custom (V2 con Let's Encrypt auto), nombre_comercial, razon_social, slogan, email_remitente_default + nombre, telefono/whatsapp_publico, direcciones_publicas jsonb, redes_sociales jsonb.

**V1 solo subdominio.gaessoft.com.** Dominio propio con SSL automático = V2.

## Bloque B — `config_localizacion`
Moneda_default (MXN|USD), moneda_secundaria_visible, fuente_tipo_cambio (banxico_dof), idioma_default (es-MX|en-US) + idiomas_disponibles array, zona_horaria, formato_fecha (DD/MM/YYYY default MX), formato_hora 24h, separadores miles/decimal, decimales_precio (2) y decimales_cantidad (3 — granel), redondeo_precio (matematico|arriba|abajo|cinco_centavos|cincuenta_centavos), pais_default MX.

## Bloque C — `config_pos`
Modo_input (touch|click|ambos), vista_default (grid|lista|favoritos), atajos_teclado_global jsonb (override default GaesSoft), layout_teclas_rapidas jsonb (botones grandes top productos por sucursal), sonidos, mostrar_imagen_producto, pedir_cliente/vendedor_default, mostrar_inventario_disponible_pos, **politica_sin_stock**, tema (claro|oscuro|auto), idle_timeout_min.

**Política sin stock cerrada por vertical:**
- Retail/Abarrotes: `advertir_continuar` (cajero decide)
- Salud (vet/humano): `bloquear` (medicamentos controlados)

## Bloque D — `config_pagos`
Métodos_aceptados array (efectivo|tarjeta|transferencia|vale|fiado|puntos_lealtad|recarga), terminales_integradas jsonb (clip|mercadopago_point|stripe_terminal|manual + creds), propinas_activas (**default OFF**, default cambia a ON si vertical Restaurantes V2), propinas_sugeridas [10,15,20], redondeo_efectivo, requerir_corte_z_diario (default ON), alerta_efectivo_caja_max, vuelto_max_alerta, permitir_pago_mixto (default ON), moneda_extranjera_aceptada jsonb.

## Bloque E — `config_inventario`
Alerta_stock_minimo, canales_alerta (email|whatsapp|push), destinatarios array user_ids, multi_almacen_por_sucursal, transferencias_requieren_aprobacion, conteo_ciclico_frecuencia, ajustes_requieren_motivo (default ON), tolerancia_diferencia_conteo_pct, **valoracion_inventario default `promedio_ponderado`** (estándar MX) — PEPS/UEPS disponibles para tenant que lo pida.

## Bloque F — `config_numeracion_interna` (NO fiscal)
Por tipo_documento (venta_pos|cotizacion|pedido|apartado|devolucion|transferencia|orden_compra|nota_inventario|cita|consulta), prefijo libre (VTA-/COT-), folio_inicial/actual, ceros_relleno (5 dígitos = VTA-00001), reset_anual, por_sucursal (cada sucursal su counter). Diferente de 4.19 (CFDI series fiscales SAT).

## Bloque G — `plantillas_impresion`
Tipo (ticket_58mm|ticket_80mm|ticket_digital_qr|factura_a4|nota_credito_a4|cotizacion_a4|pedido_a4|apartado_a4|etiqueta_producto|etiqueta_envio|corte_caja_xz|receta_medica_humano|receta_medica_vet|consulta_resumen|comprobante_pago_recibido|recibo_servicios_humanos), nombre, contenido_html (handlebars), css, variables_disponibles array, modo (default_gaessoft|basada_en_default|tenant_propia), preview_pdf_url, default_para_tipo, sucursal_id (null = global tenant).

**V1: pre-hechas GaesSoft + edición HTML/CSS** para tenant técnico/partner. **V2: editor visual drag-drop.**

## Bloque H — `config_eventos_transaccionales`
Mapping evento → plantilla. Eventos: venta_completada|apartado_creado|apartado_recordatorio|cita_confirmada|cita_recordatorio_24h|cita_recordatorio_2h|cita_cancelada|corte_z_email|receta_emitida|resultado_lab_listo|factura_emitida|nota_credito|renovacion_suscripcion_proxima|bienvenida_cliente|recordatorio_pago_cxc|stock_minimo_alcanzado|comision_calculada. Canal (whatsapp|email|sms|push), plantilla_id (fk 4.18), activo, copia_a_email_admin, ventana_envio_horas, frecuencia_max_dia.

## Bloque I — `modulos_tenant`
Modulo (pos_base|abarrotes|recargas_tae|salud_vet|salud_humana|hospitalizacion_n3|mayoreo_b2b|portal_doctoralia|telemedicina|partner_contador|ecommerce|api_externa|webhooks|multi_sucursal|multi_almacen|multi_rfc|analitica_avanzada|ai_extra_creditos), activo (calculado desde subs 4.1 + addons), origen (plan_base|addon|trial|cortesía), limite_uso jsonb (ej {sucursales: 5} en Business), uso_actual jsonb, vence_at (addons temporales), activado_at + activado_por.

## Bloque J — `feature_flags_tenant`
Flag_key slug, valor jsonb, scope (todos_usuarios|usuarios_listados|porcentaje_rollout), usuarios_incluidos array, porcentaje, activado_por (gaessoft_admin|tenant_admin), expira_at, descripcion, requiere_disclaimer bool.

**Regla:** GaesSoft activa core/seguridad; tenant puede activar betas opt-in con disclaimer obligatorio si `requiere_disclaimer=true`.

## Bloque K — Webhooks salientes (V1)

`webhooks_tenant`: url_destino, secret_hmac (encriptado), eventos_suscritos array (venta.creada|venta.cancelada|factura.timbrada|factura.cancelada|cita.creada|cita.confirmada|cliente.creado|cliente.actualizado|inventario.bajo|corte_z.cerrado|cxc.pagada|cita.no_show), timeout_seg (default 10), retry_strategy (exp 5 intentos: 1s/5s/30s/5min/1h), ultimo_resultado/intento_at/status_code.

`webhooks_logs` append-only, **retención 30 días**: webhook_id, evento, payload jsonb, response_status/body, intentos, timestamps.

## Bloque L — API keys (V1)

`api_keys_tenant`: nombre, key_prefix visible (gpk_xxxx), key_hash sha256, key_secret_encrypted, scopes array (read_products|write_products|read_sales|write_sales|read_inventory|write_inventory|read_clients|write_clients|read_invoices|read_appointments), creada_por, ultimo_uso_at + ultima_ip, rate_limit_rpm (default 60), ip_whitelist array (vacío = todas), expira_at, activa, revocada_at/por.

## Bloque M — Backups

`config_backups_tenant`: frecuencia (diaria default Pro+|semanal), hora_ejecucion (3am tz tenant), retencion_dias (30 Pro / 90 Business / 180 Enterprise), **destino solo `s3_gaessoft` V1** (BYO S3 V2), ultimo_backup_at + tamano_mb + status, descargas_manuales_mes_count + limite_descargas_mes (3 default), alerta_email_falla.

`backups_ejecuciones`: fecha, tamano_mb, status (ok|fallo|parcial), archivo_url (URL firmada), hash_sha256, expira_at, tablas_backup jsonb.

## Why
La configuración por tenant es la diferencia entre un SaaS rígido y uno que se siente "hecho a medida". Webhooks + API keys V1 son cruciales porque tenants técnicos (B2B) y partners contadores los van a pedir desde día uno. Plantillas de impresión editables HTML/CSS son suficientes V1 — drag-drop se vuelve crítico cuando hay clientes no técnicos en masa. Feature flags permiten rollout controlado de features sin tocar código.

## How to apply
Configuración vive bajo `services/config/` con caché en Redis por tenant_id (TTL 5 min, invalidación por evento de update). Webhooks usan worker con cola Redis y backoff exponencial. API keys validadas en middleware con cache local (TTL 30s). Plantillas impresión renderizadas server-side con handlebars + puppeteer/wkhtmltopdf según tipo. Backups vía pg_dump nightly + lifecycle S3 con expiración auto según retencion_dias del plan.
