# GaesSoft POS — Modelo de datos 4.2 Partners y Comisiones (cerrado)

> **Estado:** Cerrado · **Fecha cierre:** 2026-04-27
> **Memoria persistente:** `project_gaes_pos_modelo_4_2_partners.md`
> **Descripción:** Schema master DB para programa de partners contadores con comisiones recurring lifetime

Sub-análisis 4.2 (Master DB · Partners y Comisiones) aprobado por Gaby. Modelo basado en patrones PartnerStack/Rewardful + adaptado a contexto MX (CFDI inverso del partner, multi-currency MXN/USD).

**Tablas master DB en este dominio:**

`partners`: id (CUID), slug único (ej: MARGARITA), legal_name, commercial_name, rfc, **cedula_profesional** validada SSa MX (única globalmente para impedir duplicados), email_principal, phone, country, currency_default, status (pending/active/suspended/terminated), level_id FK, level_calculated_at, joined_at/approved_at/terminated_at, applied_form_data JSONB, metadata JSONB.

`partner_admins`: usuarios del portal del partner. partner_id, email, password_hash, name, phone, role (owner/assistant), mfa_enabled, is_primary, last_login_at. Permite asistentes del despacho que ayudan al contador.

`partner_levels`: id (bronze/silver/gold/diamond), name, tier_order, min_active_clients (1, 5, 10, 25), commission_rate_default (0.25 default, Diamond 0.30 para nuevos), benefits JSONB (branding_simple/full, dedicated_support, monthly_training).

`partner_links` (URLs únicas trackeables): partner_id, code (MARGARITA o MARGARITA-vet), vertical_focus (NULL=general o vet/retail_mayoreo/etc), qr_code_url S3, is_active.

`partner_link_visits` (particionado mensual): partner_link_id, visitor_fingerprint anti-double-count, ip_address, user_agent, referrer_url, utm_source/medium/campaign, cookie_set_at, converted_to_tenant_id FK opcional.

`partner_referrals`: partner_id, tenant_id (UNIQUE partial WHERE attribution_locked_until IS NULL OR > NOW), link_id FK opcional, attribution_type (first_touch_link/first_touch_coupon/manual_assignment), attribution_locked_until (default NULL=lifetime), attributed_at, metadata. Una sola atribución activa por tenant (lifetime).

`partner_commissions`: partner_id, partner_referral_id, tenant_id, invoice_payment_id (FK al pago que generó la comisión), period_start/end mes que cubre, gross_amount (lo que pagó tenant), commission_rate snapshot, commission_amount, currency, **status (pending/held/payable/paid/clawed_back/voided)**, held_until (default +14 días post-payment para cubrir refunds), payout_id FK opcional NULL hasta payout, discount_attribution JSONB (split GaesSoft/partner si cupón).

`partner_payouts`: partner_id, period_start/end, total_amount, currency, payout_method_id FK, status (pending/processing/completed/failed/canceled), external_payout_id (Stripe Connect transfer ID o ID interno SPEI), failure_reason, **cfdi_received_uuid** (UUID del CFDI que partner emite a GaesSoft por sus comisiones — input fiscal nuestro), notes, processed_at.

`partner_payout_methods`: partner_id, type (spei_clabe/stripe_connect/paypal/gaessoft_credit), is_default, clabe encriptado si SPEI, bank_name, account_holder, external_account_id (Stripe Connect / PayPal email), verified_at.

`partner_branding` (Gold+ y Diamond): partner_id, logo_url, color_primary/secondary, subdomain (Diamond only: margarita.gaessoft.com), landing_message markdown, support_contact JSONB email/teléfono visible al cliente, email_signature_html.

**Reglas de negocio cerradas:**

1. **Atribución first-touch cookie 30 días default** (Diamond override 60d como benefit). Frontend landing setea `gst_partner_attr=PARTNER_ID&exp=now+30d`. Signup dentro de ese device en 30d → atribución al partner.

2. **Atribución LIFETIME forever** para primeros 50 partners (early adopter benefit). Política revisable para nuevos después si crece costo.

3. **Cálculo de comisión** vía job `process_invoice_payment` post `succeeded`: verifica `partner_referrals` activo del tenant, crea `partner_commissions` con status `pending`, `held_until = paid_at + 14 días`. Cron diario transiciona `pending → payable` cuando `held_until < now()`.

4. **Held period 14 días MX** (Stripe 7d tarjetas, OXXO/SPEI 5d — 14d cubre ambos).

5. **Refund/clawback:** si `invoice_payment` cambia a refunded antes de held_until → comisión `clawed_back`. Si después de paid (raro) → descuento del próximo payout.

6. **Comisión solo sobre recurring**: addons one-time (cfdi_100, whatsapp_1000, ai_credits_1000, onboarding $1500) NO. Subscription_items recurring SÍ. Add-ons recurrentes (sucursal extra, usuario extra, API, storage) SÍ con proporcional al pago.

7. **Niveles auto-calculados job mensual** según count(active referrals): 0=suspended (90+ días sin actividad), 1-4=bronze, 5-9=silver, 10-24=gold, 25+=diamond. Diamond = 30% en nuevos referrals (los viejos siguen 25%).

8. **Payout mínimo $500 MXN** — acumula si menor.

9. **Frecuencia payout día 1 mes** paga lo `payable` del mes anterior. Configurable por partner.

10. **CFDI inverso obligatorio**: partner emite CFDI a GaesSoft por sus comisiones. Sistema registra `partner_payouts.cfdi_received_uuid`. Job concilia con CFDIs descargados del SAT automáticamente.

11. **Cupones partner-specific** (`coupons.partner_id`): split discount entre GaesSoft y partner. V1 default 50/50, Diamond 30/70 a favor del partner. Configurable por nivel.

12. **Partner suspendido**: comisiones siguen acumulando pero payouts pausados hasta resolución.

13. **Partner terminated**: comisiones held NO se pagan. Referrals movidos a `terminated_partner_id` campo, nuevas comisiones NO se crean.

**Índices estratégicos:**
- partners(slug) UNIQUE, (cedula_profesional) UNIQUE, (status, level_id)
- partner_links(code) UNIQUE
- partner_link_visits(partner_link_id, created_at) particionado mensual
- partner_referrals(tenant_id) UNIQUE PARTIAL WHERE active
- partner_referrals(partner_id, attributed_at)
- partner_commissions(partner_id, status, held_until) para job transición
- partner_commissions(invoice_payment_id) UNIQUE
- partner_payouts(partner_id, period_start)

**Decisiones cerradas Q&A:**
- Cookie 30d default + Diamond 60d benefit
- Held 14 días
- Thresholds 1/5/10/25
- Cupón split 50/50 V1, configurable por nivel (Diamond 30/70)
- Atribución lifetime forever primeros 50 partners

**Why:** Modelo separa atribución (referral) de devengamiento (commission) de pago (payout) — los tres son entidades distintas con ciclos de vida propios. Esto permite refunds, clawbacks, niveles, y cambios de política sin tocar los registros viejos. PartnerStack y Rewardful siguen el mismo patrón.

**How to apply:** Cuando se escriba `packages/db/prisma/schema.prisma`, este es el contrato. Endpoints `/api/partners/*` para portal del partner. Webhook listener de `invoice_payment.succeeded` dispara job `process_invoice_payment`. Cron diario de transición held→payable. Cron mensual recalcula levels + ejecuta payouts.
