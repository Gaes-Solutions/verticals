# GaesSoft POS — Modelo de datos 4.7 Productos, Inventario, Precios (cerrado)

> **Estado:** Cerrado · **Fecha cierre:** 2026-04-27
> **Memoria persistente:** `project_gaes_pos_modelo_4_7_productos_inventario.md`
> **Descripción:** Tenant schema — productos, variantes, inventario multi-sucursal, motor de precios cascada, lotes/series

Sub-análisis 4.7 (Tenant Schema · Productos, Variantes, Inventario, Listas de Precios) aprobado por Gaby. Corazón operativo del producto. Patrón base = Shopify ("todo es variante") + extensiones MX.

**Tablas tenant schema:**

`categorias` (árbol jerárquico, max 4 niveles): id, parent_id FK self-ref, nombre + nombre_i18n JSONB, slug único per-tenant, descripcion, imagen_url S3, icono (emoji o nombre), color hex, orden, vertical_aplicable (retail_mayoreo/salud_vet/etc), is_visible_pos, is_visible_publico, is_active, metadata JSONB.

`marcas`: nombre, slug, descripcion, logo_url, pais_origen, is_active.

`productos` (producto base): sku_padre único per-tenant, nombre + nombre_i18n, descripcion_corta + descripcion_larga + i18n, categoria_id, marca_id, **tipo_venta (unidad/peso/volumen/tiempo/servicio)**, unidad_medida (pza/kg/g/lt/ml/m/m2/hora), unidad_medida_compra (compras por caja, ventas por pieza), factor_compra_a_venta, **tiene_variantes** (true si tallas/colores), requires_recipe (medicamentos), clave_sat + clave_unidad_sat (CFDI), **aplica_iva** + tasa_iva default 16%, **aplica_ieps** + tasa_ieps JSONB (% o cuota fija), permite_descuento, permite_fiado, permite_apartado, permite_devolucion + dias_devolucion, dias_garantia, **requires_lote** (medicamentos con caducidad), requires_serie (electrónicos), requires_balanza (granel), peso_kg + dimensiones JSONB para envíos, **vector_embedding pgvector** para búsqueda semántica + dedup IA RF-04, is_active + is_visible_publico + is_visible_b2b, archived_at, metadata.

`producto_variantes` (la unidad SKU real — toda venta es contra una variante; producto sin variantes tiene 1 default invisible): producto_id, **sku único per-tenant** (CRO001-M-NEGRO), nombre_variante (Mediano Negro, Talla 7 Verde), opciones JSONB ({"talla":"M","color":"Negro"}), precio_base, costo_promedio (calculado por movimientos), costo_ultimo, peso_kg override, imagen_url override, is_default, is_active, archived_at.

`producto_codigos_barras`: producto_variante_id, codigo, tipo (ean13/upc/corto_interno/qr), is_primary. Una variante puede tener varios barcodes.

`producto_imagenes`: producto_id o producto_variante_id, s3_key, cdn_url, orden, tipo (principal/lifestyle/detalle/lateral/back), alt_text + alt_text_i18n, metadata (color dominante palette).

`producto_atributos` (campos custom JSONB-style): producto_id, clave libre (material/garantia/origen), valor, tipo_dato (string/number/boolean/enum). Permite que cada tenant agregue campos sin migración.

`producto_tags`: producto_id, tag.

`inventario_sucursal` (TABLA MÁS LEÍDA — optimizada concurrencia): producto_variante_id, sucursal_id, **stock_actual DECIMAL(18,3)** (decimal para granel), stock_reservado (apartados, pedidos pendientes), **stock_disponible calculado** (trigger: actual - reservado), stock_minimo + stock_maximo, ubicacion (anaquel/pasillo/bodega), last_inventory_date, next_inventory_due alerta. Constraint: UNIQUE (producto_variante_id, sucursal_id).

`inventario_movimientos` (APPEND-ONLY, particionado mensual — verdad sobre cómo cambió stock): producto_variante_id, sucursal_id, **tipo_movimiento** (compra/venta/devolucion_cliente/devolucion_proveedor/ajuste_positivo/ajuste_negativo/transferencia_salida/transferencia_entrada/merma/apartado_reservado/apartado_liberado/consumo_interno), cantidad (positivo añade, negativo resta), costo_unitario en compras, referencia_tipo + referencia_id polimórfica, lote_id FK opcional, serie_id FK opcional, motivo, usuario_id, created_at.

`producto_lotes` (medicamentos vet/humano, alimentos): producto_variante_id, numero_lote, **fecha_caducidad**, cantidad_recibida, cantidad_actual (decrementa), proveedor_id FK opcional, compra_id FK, is_active.

`producto_series` (electrónicos): producto_variante_id, **numero_serie único per-tenant**, status (disponible/vendido/devuelto/garantia/reparacion), vendido_a_cliente_id + venta_id + vendido_at, garantia_hasta.

`niveles_precio_mayoreo` (Plata/Oro/Platino): nombre, descripcion, descuento_default_pct opcional, monto_minimo_compra, orden, is_active.

`listas_precios`: codigo único (PUBLICO/MAYORISTA_ORO/EMPLEADOS), nombre + i18n, **tipo (publico/mayoreo_nivel/cliente_individual)**, nivel_mayoreo_id FK opcional, cliente_b2b_id FK opcional (per-cliente), currency default MXN, vigente_desde/hasta, is_active.

`lista_precio_items`: lista_precio_id, producto_variante_id, precio DECIMAL, **precio_minimo_negociacion** (vendedor puede bajar hasta aquí), incluye_iva. Si variante no está en lista → fallback a precio_base.

**`producto_precios_escalonados` (RF-02 — 0 a 5 niveles configurables POR PRODUCTO/VARIANTE):** producto_variante_id (o producto_id), nivel 1-5, cantidad_minima + cantidad_maxima nullable para "100+", precio_unitario, is_active. Algunos productos tienen 1 nivel solo, otros 5.

**`reglas_precio` (motor de promociones RF-06, RF-07, RF-03):**
- tipo: descuento_global_por_monto (RF-06), descuento_producto (RF-07), **mayoreo_por_total_ticket (RF-03)**, descuento_categoria, descuento_cliente, bogo_compra_x_lleva_y, precio_temporada
- condicion JSONB describiendo condición
- accion JSONB describiendo acción (% descuento, monto fijo, precio override)
- prioridad entero (primer match aplica si no es stackeable)
- stackable boolean
- **excluye_productos_con_escalonado** (RF-06 excluye productos con RF-02 ya activo)
- aplica_a (pos_fisico/ecommerce/b2b/todos)
- vigente_desde/hasta, dias_semana JSONB, horarios JSONB (happy hour)
- usos_max_por_cliente + usos_total + usos_actuales counter
- is_active

`cupones_tenant` (RF-08, distinto de cupones SaaS): codigo único per-tenant, nombre, tipo (monto_fijo/porcentaje/envio_gratis/producto_gratis), valor, monto_minimo_compra, productos_aplicables JSONB, categorias_aplicables JSONB, clientes_aplicables JSONB (empty=todos), usos_max_por_cliente + usos_total + usos_actuales, vigente_desde/hasta.

**Reglas de negocio cerradas:**

1. Toda venta contra producto_variantes.id (incluso productos sin variantes con 1 default invisible)
2. Stock decimal DECIMAL(18,3) para granel (1.345 kg)
3. stock_disponible auto-calculado por trigger Postgres
4. Apartados y pedidos pendientes incrementan stock_reservado, liberados decrementan
5. Trigger en inventario_movimientos recalcula costo_promedio en compras
6. Multi-sucursal nativo: stock siempre por sucursal, total = SUM por variante
7. **Lotes FIFO en venta**: si requires_lote, sistema sugiere lote más próximo a caducar
8. Series tracking: cada unidad vendida atada a serie específica si requires_serie
9. **Motor de precios cascada (6 pasos):**
   - 1°: producto_precios_escalonados por cantidad (RF-02)
   - 2°: lista_precio_items del cliente
   - 3°: reglas_precio que matchen
   - 4°: trigger mayoreo_por_total_ticket al cerrar venta (RF-03)
   - 5°: cupón aplicado si presente
   - 6°: descuento global del cajero (con permiso)
10. Búsqueda semántica vector_embedding: job genera embedding al crear/modificar producto. POS y ecommerce buscan con pgvector

**Decisiones cerradas Q&A:**
- Patrón Shopify "todo es variante" SÍ
- categorias árbol max 4 niveles SÍ
- producto_atributos custom JSONB SÍ
- Stock decimal DECIMAL(18,3) SÍ
- Motor de precios cascada 6 pasos SÍ

**Why:** Modelo de productos es el más leído del sistema. Errores aquí = lentitud crónica. Patrón Shopify-style con variantes uniformes simplifica todo. Motor de precios cascada cubre TODOS los RFs (1, 2, 3, 6, 7, 8, 19) sin special cases. pgvector permite búsqueda semántica WOW que Eleventa no tiene.

**How to apply:** Backend: motor de precios como servicio dedicado `services/pricing-engine.ts` con cascada de 6 pasos. Triggers Postgres para stock_disponible y costo_promedio (más rápido que código aplicación). Vector embeddings generados async vía worker BullMQ post-create/update. Frontend: componente buscador con switch entre búsqueda exacta (barcode) y semántica (texto natural).
