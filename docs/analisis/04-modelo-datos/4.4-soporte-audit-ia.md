# GaesSoft POS — Modelo de datos 4.4 Soporte, Audit, Pipeline + 4.5 IA Usage (cerrado)

> **Estado:** Cerrado · **Fecha cierre:** 2026-04-27
> **Memoria persistente:** `project_gaes_pos_modelo_4_4_soporte_audit_pipeline.md`
> **Descripción:** Schema master DB para soporte tickets, audit log inmutable, CRM ligero, créditos IA

Sub-análisis 4.4 (Soporte/Audit/Pipeline comercial) + 4.5 (IA Usage) aprobados por Gaby. Cierra master DB completo. Próximo: tenant schema desde 4.6.

**Tablas master DB:**

`gaes_team_users` (4to tipo de usuarios, separado de tenant_users_admin/usuarios per-tenant/patient_master): id, email, password_hash, name, phone, role (owner/support/comercial/operaciones/desarrollo), mfa_enabled (mandatorio true para superadmin), permissions JSONB granulares, is_active, last_login_at/ip.

`support_tickets`: tenant_id, opened_by_user_id (FK polimórfico tenant_users_admin o usuarios per-tenant), assigned_to_team_user_id (NULL si bot), subject, category (bug/pregunta/solicitud_feature/billing/cancelacion/urgencia), priority (low/medium/high/urgent), status (open/pending_customer/pending_internal/resolved/closed), channel (web_app/email/whatsapp/phone), **sentiment_score IA -1.0 a 1.0**, sla_first_response_due_at + sla_resolution_due_at (basado en plan del tenant), first_responded_at/resolved_at/closed_at, resolution_summary, tags JSONB, metadata.

**SLA por plan:**
- Starter: primera respuesta <24h
- Pro: <8h
- Business: <2h
- Enterprise: <30 min

`support_ticket_messages`: ticket_id, sender_type (tenant_user/team_user/bot_ai), sender_id polimórfico, body markdown, body_html cacheado, is_internal_note (visible solo equipo), was_helpful (thumbs up/down del cliente).

`support_ticket_attachments`: ticket_id, message_id opcional, file_name, mime_type, s3_key, size_bytes.

`audit_logs` (TABLA INMUTABLE — append-only, particionada mensual, archivado a cold storage tras 24 meses con pg_partman): id UUID v7 (ordenable por tiempo), actor_type (team_user/tenant_user_admin/system/bot), actor_id, tenant_id NULL si global, **action** string descriptiva (tenant.created, tenant.suspended, subscription.changed, impersonation.started/ended, coupon.applied, partner.commission.paid, phr.record.viewed, support_ticket.responded, etc), resource_type (tenant/subscription/invoice/partner/patient/support_ticket), resource_id, before_state JSONB, after_state JSONB, **reason texto libre obligatorio para impersonations y refunds**, ip_address, user_agent, country, created_at. Triggers Postgres rechazan UPDATE/DELETE. Indexable por (actor_id), (tenant_id), (action), (resource_type+resource_id), (created_at). Cumple SOC2/ISO 27001/LFPDPPP.

`leads` (CRM ligero V1, HubSpot integrado V2): email, phone, name, company, vertical_interest, source (landing/partner/inbound_call/referral/other), partner_id FK opcional, utm_source/medium/campaign, status (new/contacted/demo_scheduled/demo_held/proposal_sent/negotiating/won/lost/nurturing), qualification_score 0-100 calculado, assigned_to_team_user_id, notes markdown, tags JSONB, last_contact_at + next_follow_up_at, lost_reason si lost, won_to_tenant_id FK opcional si convertido.

`lead_activities`: lead_id, team_user_id, activity_type (call/email/whatsapp/meeting/demo/note), subject, body, duration_minutes si call/meeting, outcome (positive/neutral/negative/no_answer).

`deals`: lead_id, tenant_id_proposed, plan_proposed, mrr_proposed, currency, probability 0-100%, expected_close_date, stage (negotiating/proposal_sent/signature_pending/won/lost), lost_reason, won_at/lost_at.

`deal_activities`: similar a lead_activities con deal_id.

`demos_scheduled`: lead_id o deal_id, team_user_id, scheduled_at, duration_minutes, meeting_link (Daily.co o Zoom), status (scheduled/held/no_show/cancelled/rescheduled), recording_url, notes_after.

**IA Usage (sub-análisis 4.5 integrado):**

`ai_credit_balances`: tenant_id, current_balance default 0, monthly_allowance (de plan: 100/1000/10000/unlimited), last_reset_at mensual, lifetime_consumed.

`ai_credit_transactions`: tenant_id, transaction_type (monthly_grant/purchase/consumption/refund_partial), amount (positivo añade, negativo consume), reason descripción, feature_key (si consumo: expediente_resumen, producto_descripcion, whatsapp_chatbot, etc.), external_purchase_id si purchase.

`ai_usage_logs` (particionado mensual): tenant_id, feature_key, model (claude-sonnet-4-6/gpt-4o-mini/whisper-1/text-embedding-3-small), input_tokens + output_tokens + total_cost_usd raw del proveedor, credits_consumed lo que cobramos, latency_ms, success boolean, error_code si fail, tenant_user_id originador, subject_type/subject_id a qué entidad sirvió. Sirve para reportes costo/ingreso, análisis features más usadas, detección abuso (tenant consumiendo 1000x lo normal).

**Reglas de negocio cerradas:**

1. Audit log inmutable garantizado a nivel DB con triggers
2. SLA por plan respetado en tickets, job alerta si cerca de vencer
3. Auto-asignación tickets: bot IA primero, si no resuelve en 5 min o cliente pide humano → escala a team_user con menos carga
4. Pipeline lead→deal→tenant: deal won crea tenant automático con plan acordado
5. Renovación mensual de créditos AI día 1: saldo se resetea a monthly_allowance (NO se acumula lo no usado del mes anterior — política simple V1)
6. Add-on de créditos via tenant_addon_purchases SE ACUMULA al current_balance y NO se pierde fin de mes
7. Soft fail si saldo insuficiente: feature IA se desactiva con mensaje claro, NO bloquea producto

**Decisiones cerradas Q&A:**
- gaes_team_users separado: SÍ (4to tipo de usuarios)
- Audit log particionado mensual + archivado 24 meses: SÍ
- Créditos AI no acumulables mes a mes (mensuales reset), add-ons SÍ acumulan: SÍ
- CRM ligero V1 con leads/deals/activities, HubSpot V2: SÍ

**Why:** Cierra master DB con dominio operativo (soporte + comercial + observabilidad de IA). Audit log es base legal/compliance (sin él no hay SOC2 ni ISO ni venta enterprise). CRM propio evita pagar HubSpot $200+ USD/mes con 5 clientes.

**How to apply:** Endpoints `/api/admin/support/*` para tickets desde admin panel. Bot IA con LangChain + RAG sobre docs como primer respondiente. Job programado cada 5 min mide SLA breach. Job mensual día 1 resetea ai_credit_balances. Webhook listener de OpenAI/Anthropic dispara registro en ai_usage_logs.
